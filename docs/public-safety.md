# Public Safety

The public AnarchI portfolio should demonstrate capability without exposing the machinery that makes the systems defensible.

## Safe To Publish

- Architecture summaries
- Typed interfaces
- Toy fixtures
- Presentation-safe dashboards
- Deterministic patterns without private weights
- Public brand language
- Test harnesses with synthetic data

## Keep Private

- Credentials and tokens
- Runtime memory databases
- Browser profiles and session caches
- Live account state
- Wallet keys or seed material
- Private customer or user records
- Unreleased CERBERUS decision chains
- Exact scoring formulas for adversarial environments

## Rule

Public repos should explain what the system is and why it matters. Private repos should preserve the exact machinery that creates advantage.
